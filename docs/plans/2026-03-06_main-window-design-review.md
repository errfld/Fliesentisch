# Main Window Design Review

## Goal
- Create three distinct visual directions for the app's main window.
- Keep the concepts centered on the real-table fantasy: spotlighting, whispers, presence, and shared play surface.
- Mount the concepts at `/1`, `/2`, and `/3` for direct comparison.

## Approach
- Build the concepts as isolated review routes so the live room flow remains intact.
- Reuse one shared set of sample room data to compare information architecture across the three directions.
- Give each route a different visual language rather than small cosmetic variations.

## Validation
- Run frontend checks after implementation.
- Open the concepts in a real browser and confirm layout, responsiveness, and visual differentiation.
